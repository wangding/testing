第 11 章：Git 入门

1. 浏览参考资料

要求：

  • 阅读维基百科的 GitHub 词条；
  • 阅读百度百科的 GitHub 词条；
  • 浏览 GitHub 年度报告；

2. 注册 GitHub 账户

请注册自己的 BitBucket 账户，具体要求如下：

  • 要有自己的真实姓名；
  • 要有自己的真实头像；
  • 要有自己的职业信息；
  • 要有自己的单位信息；
  • 要有自己的位置信息；

3. GitHub 社交

要求：

  • Follow（关注）王顶
  • Star（收藏）testing 项目
  • Follow（关注）其他人（可以是王顶关注的人）
  • Star（收藏）其他项目（可以是王顶收藏的项目）
  • Watch（关注）courses 项目

4. GitHub 挖宝

要求：

  • 搜索活跃的大牛，关注他
  • 查找感兴趣的项目，收藏它

5. GitHub 仓库

要求：

  • 浏览 testing 仓库；
  • 创建 demo 仓库；
  • 为 demo 仓库启用 GitHub Pages 设置；
  • 在 demo 仓库中操作；
  • 删除 demo 仓库；

6. GitHub 工具

要求：

  • 百度搜索 Git 并下载；
  • 安装 Git 工具；
  • Git Bash 中运行 git --version 验证安装是否成功；
  • 配置 Git Bash 环境；
  • 配置 Git 初始参数：姓名和邮箱；
  • 练习简单的 Linux Bash 命令；

7. 本地练习 Git

要求：

  • 在自己电脑的 C 盘根目录（或者 linux 的家目录），建立文件夹：hello-git；
  • 在 hello-git 中初始化仓库；
  • 在 hello-git 中创建 README.md 文件，文件内容随意；
  • 把 README.md 文件，提交到本地仓库中；
  • 在自己的 GitHub 账户中创建 hello-git 空仓库；
  • 把自己本地的 hello-git 仓库与远程（上一步）hello-git 仓库关联上；
  • 把本地的 hello-git 仓库中的内容 push（同步）到远程 hello-git 仓库中；
  • 在 GitHub 的 hello-git 仓库中，验证存在文件：README.md；
  • 在 GitHub 上修改 README.md 文件；
  • 在本地的仓库中把上一步的修改 pull 拉下来；

8. 在线练习 Git

要求：

  • 克隆 learnGitBranching 仓库
  • 阅读 README.md
  • 在线练习网址：https://learngitbranching.js.org/
  • 完成基础 4 关的在线练习

9. 闯关练习 Git-it

要求：

  • 安装 NodeJS；
  • 命令行安装闯关学习软件：npm i -g git-it
  • 打通 11 关；
